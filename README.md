# 🌿 HothouseSim

---

## 🚀 Что реализовано

* **Grid System (Сетка симуляции):** Кастомная логика управления ячейками, поддерживающая состояния: *Свободно*, *Посажено*, *Готово к сбору*, *Требует полива*.
* **Механика роста:** Система стадий развития культур (Tomato, Cucumber) с динамической зависимостью от уровня влаги и времени.
* **Persistence Subsystem:** Система сохранений, обеспечивающая полную сериализацию состояния сетки и прогресса игрока в именованные слоты.
* **Модульный UI:** Динамические контекстные меню, HUD со статистикой в реальном времени и система взаимодействия через кастомный `PlayerController`.

---

## 🏗 Организация данных

Проект разбит на логические слои для обеспечения слабой связности (**Low Coupling**):

* **Core:** Движок симуляции и базовая математика сетки.
* **HothouseSettings:** Важные настройки для преокта лежат внутри
* **CropData:** Настройки саженцев вынесены в отдельный конфиг. Можно задавать новые типы и желаемые свойства. 
* **Models:** "Чистые" POD-структуры данных (`Crops`, `Cells`, `Stats`), полностью отделенные от игровой логики.
* **Subsystems:** Глобальные сервисы (`GridSubsystem`, `PersistenceSubsystem`) для управления глобальным стейтом, доступные из любой точки кода.
* **UI/View:** Разделение на логику кнопок и контроллеры отображения ячеек (`CellView`), работающие по принципу, близкому к **MVVM**.

---

## 🛠 Почему выбрана такая структура?

1.  **Scalability:** Использование `UGameInstanceSubsystem` позволяет легко расширять функционал (погода, рынок, экономика), не перегружая `GameMode` или `Level Blueprint`.
2.  **Data-Driven Design:** Параметры культур вынесены в отдельные структуры данных. Это позволяет геймдизайнеру менять баланс (скорость роста, потребление воды) без пересборки кода.
3.  **Performance:** Все тяжелые вычисления (итерации по сетке, логика роста) инкапсулированы в C++, оставляя Blueprints только роль "клея" для визуальных эффектов.

## 📅 Что можно улучшить)

* **Разгрузить GridSubsystem** Хоть текущая реализация и приемлема для прототипа, но дальнейшее масштабирование требует выделение отдельной система интеракции с ячейками. Также следует доработать GridSimulation для учета уровня освещения саженца и последующее применение эффекта. Таким образом напрашивается система эффекторов.
* **Разработка системы интеракции для работы с саженцами** В данные момент есть только два действия: посадить саженец и полить. Но сам примененый подход довольно топорный. Также в данный момент эта логика лежит внутри HothousePlayerController, что в свою очередь излишне.
* **Также стоит выделить менеджер сохранений в отдельную сущность дял унификации этой логики**
* **Почистить UI дял полноценного переезда на MVVM**
* **Пробежаться по неймингу файлов в папке Models** Сейчас есть проблема с именованием некоторых файлов, которые именуются моделью, но по своей сути являются обычными дата классами 
* **Различные правки UI**

## 💻 Инструкция по запуску

1.  Убедитесь, что установлен **Unreal Engine 5.5**.
2.  Склонируйте репозиторий в папку проекта.
3.  Нажмите правой кнопкой на файл `.uproject` -> **Generate Visual Studio project files**.
4.  Скомпилируйте решение в вашей IDE (**Rider**, **Visual Studio** или **CLion**).
5.  Запустите проект и откройте карту `Minimal_Default`.
6.  Если нужно что-то моменять в настройках проекта, то следует найти HothouseSettings и задать нужные настройки.
7.  Стартуем сцену.
